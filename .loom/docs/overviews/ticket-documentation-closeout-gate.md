# Ticket documentation closeout gate

Ticket completion now requires an explicit documentation disposition.

## What must be recorded

Significant work must either cite the governed docs that changed, create or update the required doc records, supersede or archive outdated docs, or record a waiver reason that truthfully explains why the explanatory surface did not change.

## Why this gate exists

The docs layer is the durable explanatory surface for accepted system reality. Closing execution work without recording the documentation impact leaves the corpus stale and untrustworthy.

## Expected evidence

Completion evidence should point at the specific governed doc records that changed or the explicit waiver recorded on the ticket.
