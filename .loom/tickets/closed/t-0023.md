## Summary
Wire worker records into the surrounding Loom layers through explicit refs, packets, and update hooks so worker execution is traceable everywhere it matters without turning workers into a shadow replacement for tickets, plans, Ralph, critique, or docs.

## Context
Why this exists: the worker substrate only fits Pi Loom if it composes cleanly with the existing stack. This ticket is about architectural honesty: workers add workspace-execution detail, but they must not split provenance or duplicate the responsibilities already owned by tickets, plans, Ralph, critique, or docs. The result should make cross-layer tracing obvious and deterministic.

What a newcomer should understand before touching code:
- Every worker must link to at least one ticket.
- Workers may also link to plans, specs, initiatives, research, critiques, Ralph runs, and docs.
- Tickets remain the live execution ledger; worker state complements ticket truth rather than replacing it.
- Ralph may launch or observe worker-manager execution, but worker internals must not be absorbed into Ralph run state.
- Critique and docs should reference worker evidence via links and packets, not by flattening worker internals into their own canonical records.

Likely files and areas touched:
- worker models/render/store code for cross-layer refs
- ticketing integration points for journal/external refs/checkpoints
- plan and possibly Ralph packet/dashboard integration points
- tests proving rediscovery/tracing from adjacent layers

Important edge cases:
- plan links a ticket set and later needs to rediscover the workers spawned for those tickets
- Ralph launches a manager process but remains run-centric rather than worker-centric
- critique references a consolidation outcome without worker records pretending to be critique artifacts
- docs updates should use worker evidence as input without becoming operational state

Dependencies and sequencing:
- Depends on worker model, checkpoint/telemetry, and consolidation tickets.
- This ticket is late in the sequence because it must integrate with real worker records and outcomes, not placeholders.

## Plan
1. Add explicit cross-layer refs in worker records and any needed helper queries for tracing workers from tickets/plans/specs/initiatives/research/critiques/Ralph/docs.
2. Update ticketing integration so important worker checkpoints, approvals, and consolidation outcomes can be journaled or externally referenced without replacing ticket truth.
3. Add plan/Ralph packet and summary hooks that can point at workers while keeping those layers bounded.
4. Ensure critique/docs interactions consume worker evidence through references and packets rather than duplicated state.
5. Add tests that prove bidirectional provenance and prevent worker artifacts from becoming a shadow ticket or Ralph system.
