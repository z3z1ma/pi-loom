## Summary
Dashboard artifacts are rewritten with volatile `generatedAt` timestamps even when no durable project truth changes, causing meaningless git diffs.

## Context
Evidence:
- `packages/pi-plans/extensions/domain/dashboard.ts`, `packages/pi-critique/extensions/domain/dashboard.ts`, `packages/pi-docs/extensions/domain/dashboard.ts`, `packages/pi-ralph/extensions/domain/dashboard.ts`, and `packages/pi-research/extensions/domain/dashboard.ts` all stamp `generatedAt: currentTimestamp()`.
- Their stores unconditionally rewrite `dashboard.json` during normal reads/updates.
- The current repo already shows dashboard-only timestamp churn in checked-in `.loom` data.
These dashboards read like observability rollups, not canonical user-authored records.

## Plan
1. Decide whether `dashboard.json` is canonical state or a derived cache.
2. If canonical, remove purely generated timestamps and avoid rewriting when semantic content is unchanged.
3. If derived, stop tracking it in git and document regeneration behavior.
4. Add tests that repeated no-op reads/updates do not dirty dashboards.

## Notes


## Verification
Two identical reads/updates in a clean repo should not produce dashboard diffs when no durable state changed.

Removed volatile dashboard generation timestamps and verified stable dashboard shapes through updated tests plus `npm run typecheck`, `npm run lint`, and `npm test`.
