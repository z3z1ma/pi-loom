## Summary
The `pi-ralph` README still describes major command/tool/runtime/test surfaces as future work even though they are already implemented.

## Context
Evidence:
- `packages/pi-ralph/README.md` says command surface, AI tools, launch adapters, verifier integration, and tests are follow-on work.
- `packages/pi-ralph/extensions/index.ts` and `packages/pi-ralph/extensions/tools/ralph.ts` already register `/ralph`, `ralph_launch`, and `ralph_resume`.
- `packages/pi-ralph/__tests__/index.test.ts` and `packages/pi-ralph/__tests__/tools.test.ts` cover those surfaces.
This documentation drift understates the real contract and makes maintainers treat landed behavior as aspirational.

## Plan
1. Rewrite the README status section around the surfaces that are actually shipped today.
2. Move any remaining future work into a bounded "remaining gaps" section.
3. Cross-check the README against the package index/tools/tests so the documented contract is current.
