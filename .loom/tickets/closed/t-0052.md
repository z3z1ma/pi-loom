## Summary
Move `pi-critique`, `pi-ralph`, and `pi-docs` onto the shared storage contract while preserving repo-materialized main docs markdown bodies and deliberate review packet/projection behavior.

## Context
Follow-up to the foundational storage substrate slice (`t-0044` through `t-0049`). These packages still persist canonical machine state in repo files and reopen neighboring artifacts directly. This ticket migrates the review/orchestration/documentation layers onto shared storage while preserving the human-facing docs carve-out.

In scope:
- `packages/pi-critique/`
- `packages/pi-ralph/`
- `packages/pi-docs/`
- integration with `@pi-loom/pi-storage`

Important carve-out:
- keep the main docs markdown body/bodies repo-materialized and grep-friendly
- keep critique/Ralph packets and other review surfaces only as deliberate projections where they still earn their keep
- move machine metadata/state/history into shared storage unless a specific projection is justified

## Plan
1. Add storage adapters for critique, Ralph, and docs on top of `@pi-loom/pi-storage`.
2. Preserve repo-materialized docs markdown bodies while moving docs metadata/state into shared storage.
3. Replace canonical file assumptions for critique/Ralph runs, findings, packets, and dashboards with shared storage queries plus explicit projections.
4. Update focused tests to prove the new canonical source of truth and preserved repo-materialized docs behavior.
